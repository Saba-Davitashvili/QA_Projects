# [BUG-017] View Request Button Unresponsive After Returning from Gardener Profile

## Metadata
- **Project**: On-Demand Gardening & Landscaping Service Platform
- **Component**: Navigation / Request Flow
- **Type**: Bug / UI
- **Severity**: High
- **Priority**: High
- **Status**: Open

## Environment Details
- **Platform**: Desktop (Windows 11, Opera GX 131)
- **App Context**: Gardener Matching List Screen

## Preconditions
User is logged in.
User has successfully submitted a service request.

## Steps to Reproduce
1. Click the "Profile" button on any gardener card.
2. Click the browser "Back" button to return to the gardener list.
3. Click the "View Request" button.

## Actual Result
The "View Request" button is completely unresponsive and fails to trigger any action.

## Expected Result
The user is successfully navigated to the details page of their submitted request.
