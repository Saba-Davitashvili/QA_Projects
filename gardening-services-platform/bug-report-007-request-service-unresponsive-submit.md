# [BUG-015] Request a Service Button Becomes Unresponsive Post-Submission

## Metadata
- **Project**: On-Demand Gardening & Landscaping Service Platform
- **Component**: Navigation / Request Flow
- **Type**: Bug / UI
- **Severity**: High
- **Priority**: High
- **Status**: Open

## Environment Details
- **Platform**: Desktop (Windows 11, Opera GX 131)
- **App Context**: Main Dashboard

## Preconditions
User is logged in.
User has successfully submitted a service request (steps 1–5).

## Steps to Reproduce
1. Click on the "Request a Service" button in the top-left corner.

## Actual Result
The button is completely unresponsive, and no action is triggered.

## Expected Result
The button should remain active, allowing the user to initiate a new request flow.
