# [BUG-069] Suggestion to Support Drag and Drop Interactions for Photo Uploads

## Metadata
- **Project**: On-Demand Gardening & Landscaping Service Platform
- **Component**: UI / Usability Suggestion
- **Type**: Improvement / UX
- **Severity**: Low
- **Priority**: Medium
- **Status**: Open

## Environment Details
- **Platform**: Desktop Browsers
- **App Context**: Photo Upload Components

## Preconditions
The current photo upload mechanism requires users to click and manually browse their file explorers.

## Steps to Reproduce
1. Navigate to any file or photo upload field.

## Actual Result
Users cannot drag and drop image files directly into the upload area; they must use manual file explorer selection.

## Expected Result
Implement a standard drag and drop zone parallel to the manual click-to-browse, providing visual highlight states when a file is hovered over the drop zone.
